Modular External Library for In Situ Sensitivity Analysis

Melissa code will be available on this github repo before Supercomputing 2017

Overview

Melissa is an in situ solution for sensitivity analysis. It implements iterative algorithms to compute spatio-temporal statistic fields over results of large scale sensitivity studies. Melissa rely on a client/server architecture, composed of three main modules:

  • Melissa Server: an independent parallel executable. It receives data from the simulations, updates iterative statistics as soon as possible, then trow data away.
  • Melissa API: a shared library to be linked within the simulation code. It mainly transmit simulation data to Melissa Server at each timestep. The simulations of the sensitivity analysis become the \textit{clients} of Melissa Server.
  • Melissa Master: A Python script in charge of generating and managing the whole global sensitivity analysis.
Simulation outputs are never stored on disc. They are sent by the clients, and consumed by the server. Melissa is built around two key concepts: iterative statistics algorithms and asynchronous client/server model for data transfer. This allows to compute statistics maps on every mesh element for every timestep.
