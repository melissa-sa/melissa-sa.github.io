Modular External Library for In Situ Sensitivity Analysis

Melissa code will be available on this github repo before Supercomputing 2017

Overview

Melissa is an in situ solution for sensitivity analysis.
It implements iterative algorithms to compute spatio-temporal statistic fields over results of large scale sensitivity studies. Melissa is built around two key concepts: iterative statistics algorithms and asynchronous client/server model for data transfer. Simulation outputs are never stored on disc. They are sent by the simulations to a parallel server, which agregate them to the statistic fields in an iterative fashion, and then trow them away. This allows to compute statistics maps on every mesh element for every timestep on a full scale study.

Melissa Workflow

Melissa rely on a client/server architecture, composed of three main modules:

  • Melissa Server: an independent parallel executable. It receives data from the simulations, updates iterative statistics as soon as possible, then trow data away.
  • Melissa API: a shared library to be linked within the simulation code. It forwards simulation data to Melissa Server. The simulations of the sensitivity analysis become the clientsof Melissa Server.
  • Melissa Master: A Python script in charge of generating and managing the whole global sensitivity analysis.
