Getting Started

Source Code

Download Melissa sources here.

Dependencies

  • Cmake
  • MPI, OpenMP: for melissa parallel server, a sequential server will be built if not available
  • Python 2
  • ZeroMQ: version 4.1.5 or above - cmake can download and install it if cmake option tuned on
    -DINSTALL_ZMQ=ON
  • Fortran Compiler, PythonLibs, Numpy: for the exemples

CMake Options

  • -DCMAKE_INSTALL_PREFIX (default: '../install')
    • Melissa install directory.
  • -DBUILD_WITH_MPI (default: ON)
    • Enable MPI.
  • -DBUILD_WITH_OpenMP (default: OFF)
    • Enable OpenMP for Melissa Server.
  • -DINSTALL_ZMQ (default: OFF)
    • Enable CMake to install ZeroMQ.
  • -DBUILD_DOCUMENTATION (default: OFF)
    • If Doxygen is found, build the Doxygen documentation.
  • -DBUILD_EXAMPLES (default: ON)
    • Compile examples.
  • -DBUILD_TESTING (default: ON)
    • Build Melissa tests. They can be run with make test or ctest.
  • -DBUILD_WITH_PROBES (default: OFF)
    • Add some debug outputs to Melissa server.

Compile and Install

  • Compilation, installation and environment variable setting from the Melissa root directory: 
    
mkdir build
cd build
cmake ../source -DINSTALL_ZMQ=OFF
make
make install
source ../install/melissa_set_env.sh

    Testing

    The examples are built if CMake finds a Fortran compiler and if you enabeled the BUILD_EXAMPLES option. The examples are installed in install/share/melissa/examples. We will use the heat equation solver example to test the installation. In the install/share/melissa/examples/heat_example directory, you will find the solver in the base directory, and several sensitivity analysis preconfigures for different batch schedullers. For testing purposes, from the base directory run: 
    
  melissa_launcher .
    It will launch a pre-configured sensitivity analysis on your local machine. To see more details about this example, check the use case documentation.
